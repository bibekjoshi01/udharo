# उधारो (Udharo) – Simple Nepali Ledger

Udharo is a Nepali-first, offline-first khata app that helps shopkeepers track customers, credits, payments, reminders, and generated PDFs. The app keeps data on-device (SQLite), supports reports, secure lock screens, backups, and bilingual labels (Nepali + English).

## Prerequisites

1. **Node.js & npm** – install via [nodejs.org](https://nodejs.org/) (Node 18 or newer is recommended for Expo SDK 54). npm is bundled with Node.
2. **Expo CLI** – run `npm install -g expo-cli` if you prefer a global helper; all commands below work with `npx` as well.
3. **Android Tooling** – Android Studio + SDK for `npx expo run:android` and local builds.
4. **iOS Tooling** – Xcode 15+ and command line tools (macOS only) for `npx expo run:ios` or Xcode builds.
5. **EAS CLI** – install with `npm install -g eas-cli` when preparing cloud or local builds.

## Getting Started (Local Development)

```bash
npm install
```

### Run the Metro Bundler

```bash
npx expo start
```

Options while the bundler is running:

- `npm run android` (alias for `expo run:android`) – builds and installs on a connected emulator/device.
- `npm run ios` – builds and runs on simulator (macOS only).
- `npm run web` – launches the web version of the app.

If you want to use the Expo Dev Client, run `npx expo prebuild --platform android` once, then `expo run:android` as normal.

## Building for Distribution

### Cloud builds with EAS

1. Authenticate: `npx eas login` (create an Expo account if you do not have one).
2. Select the build profile you need from `eas.json`. The repo exposes:
   - `development`: internal development client build (debug + dev client enabled).
   - `preview`: internal distribution APK/AAB for QA (production env variables).
   - `production`: Play Store / App Store build (production env variables).
3. Example commands:

```bash
npx eas build -p android --profile preview
npx eas build -p ios --profile production
```

### Local APK/AAB (Android only)

```bash
npx expo prebuild --platform android
eas build -p android --profile preview --local
```

Local builds assume Android SDK + NDK are configured and accessible to the shell.

## App Structure Highlights

- `App.tsx` / `src/` – entry point and feature screens (customers, credits, payments, reports).
- `assets/` – fonts, icons, and static resources.
- `android/`, `ios/` – native projects generated by Expo prebuild.
- `docs/release-guide.md` – release checklist, migration notes, and Play Store flow.

## Troubleshooting

- **Metro fails after dependency changes**: run `npx expo start -c` to clear Metro cache.
- **Firebase config missing**: add the correct `GoogleService-Info.plist` (iOS) and `google-services.json` (Android) before building.
- **EAS build credentials**: credentials are managed remotely. Use `eas credentials` to inspect or rotate if needed.

For more release details (versioning, migrations, Play Store steps), see `docs/release-guide.md`.
